# 调研任务（第二轮）：LLM 生成儿童线稿的最佳媒介形态

## 背景变化

上次我问的是"怎么让 LLM 写出更好的 SVG"。但经过反思，我想问一个更根本的问题：

**如果我们不执著于"让 LLM 直接生成 SVG 文本"这个路径，还有哪些方式能让 LLM 帮我们生成儿童黑白线稿，同时保持"线条一根根自己画出来"的动画效果？**

## 项目约束（不变的）

- **目标用户**：父母+4岁孩子，对着手机口述故事，每说完一句话生成一幅画，所有画连起来变成绘本
- **视觉风格**：黑白线稿，手绘感，不要彩色，不要照片
- **动画**：必须保持"铅笔在纸上自己画出来"的逐笔描边感——这是产品的灵魂
- **延迟**：单幅画<8秒（孩子能等的上限）
- **成本**：不要自建模型、不要本地 GPU（纯 API 调用方案）
- **复杂度**：简单场景，3-4 岁孩子的认知水平（恐龙、月亮、花朵、蝴蝶、小猫）

## 已知的死路（不用再建议的）

以下路径我们已经试过并排除了：
- **LLM 直接写 SVG 代码**（DeepSeek-flash / deepseek-chat / 豆包全系都试过）→ 几何崩、太简陋、没有层次。LLM 本质上是语言模型，不是 CAD 软件。
- **豆包 seed-2.0 系列**（PRO 是视觉模型 31-45s 超时，LITE 文本模型 25s 超时）→ 太慢。
- **DeepSeek Reasoner + JSON mode** → 之前不支持（现在支持了，但质量不一定好）。

## 我想探索的方向（开放式的）

### 方案 A：LLM 当"美术指导"，确定性代码当"画师"
- LLM 只负责描述"画什么、在哪、多大、什么动作"（结构化 JSON），一段确定性代码负责把这些描述转成实际的线条
- 类似：LLM 说"在画布中心有个椭圆代表身体，上方有个圆代表头"，代码画出来
- 问题：确定性代码的"画师"要怎么实现？用模板库？用程序化生成（procedural generation）？能用 p5.js 或 Paper.js 这类库吗？
- 这个方向有没有成熟的开源方案或参考项目？

### 方案 B：图片生成 + 线稿提取
- 用 AI 图片生成模型（DALL-E / Stable Diffusion / Midjourney）先生成一张高质量插图
- 然后用边缘检测、线稿提取等技术把彩色图转成黑白线稿
- 再用某种方式把线稿转成可描边动画的格式
- 问题：整体延迟（生成+提取+转格式）能控制在 8 秒内吗？有没有成熟的线稿提取 pipeline？
- 提取出的线稿质量怎么样？能保留手绘感吗？

### 方案 C：程序化/参数化绘图（Procedural Art）
- LLM 输出一段 JavaScript 绘图程序（像 p5.js 或 Processing 那样），而不是 SVG 标签
- 优势：LLM 对代码的理解比纯 SVG 坐标好很多（代码是它训练数据里的主力）
- 程序执行后可以生成 Canvas 动画或 SVG 输出
- 问题：有没有人做过"LLM → p5.js → 儿童插画"的实验？效果怎么样？

### 方案 D：模板式拼贴
- 预制一套角色/场景的 SVG 组件库（恐龙身体、恐龙腿、月亮、星星、花朵等）
- LLM 的职责缩小为"从组件库里选零件 + 排列 + 描述动作"
- 类似于"乐高式"组装，保证每个零件都好看，LLM 只管创意组合
- 问题：表达力会不会太受限制？有没有类似的"LLM + 组件库"混合生成的开源方案？

### 方案 E：描摹（Tracing）或草图细化
- LLM 生成一个粗糙的"骨骼框架"（比如 10 个关键坐标点），确定画面的大致布局
- 然后一段确定性算法基于这些坐标点生成流畅的 SVG 路径（类似贝塞尔曲线插值）
- 类似于"你画我猜"里的关键词→简笔画的过程
- 问题：这个方向有没有现成的算法/模型可以用？比如 sketch-rnn 或类似的东西？

### 方案 F：Canvas / HTML5 逐帧脚本
- LLM 生成的不是 SVG，而是一段"绘图指令序列"——类似于 logo 语言的那种"前进 50 像素，右转 30 度，画圆"
- 浏览器逐条执行指令，每条执行一个动画步骤，天然产生"一笔一画"的效果
- 这实际上回到了最原始的"乌龟绘图"模型——但对 LLM 来说比 SVG 容易吗？

### 方案 G：ASR + 图像生成一体化
- 用单一的多模态模型（如 GPT-4o）同时做语音识别+故事理解+画图，一步完成
- 理论上模型内部对场景的理解不需要经过"SVG 文本"这个中间态
- 问题：目前有没有多模态模型能输出"可做逐笔动画"的格式？还是只能输出 PNG？

### 方案 H：其他我没想过的
- 如果你知道任何其他"让生成式 AI 参与线条创作→最终出线稿动画"的路径，请尽情提出。我不需要方案"一定可行"，我只需要知道"有人在这个方向上做过、效果大概怎么样"。

## 调研要求

1. **每个方案都请给出**：该方案的工作原理（100 字内）、是否有人做过类似的（给项目名或论文标题）、延迟/成本是否在可接受范围内、对 4 岁孩子画能接受的复杂度来说是否过度设计。

2. **不要被"SVG 是唯一出路"这个假设限制住**。如果某个方案最终输出是 Canvas、WebGL、甚至 GIF，只要它能做"线条一笔笔画出来"和"黑白手绘风"，就值得考虑。

3. **给出一个综合推荐**：基于你的调研，哪条路径在"质量（像不像孩子的画）/ 延迟（<8秒）/ 成本（纯 API）/ 动画可行性"四个维度上平衡得最好？如果最优方案需要换供应商（比如从 DeepSeek 换到 OpenAI），也请实说。

4. **如果某条路"理论上很好但工程上很重"**，请标注出来——我没有大团队，也没有 GPU 集群。

5. **上一个调研报告里关于 SVG 生成的那些发现（scene plan 分层、本地编译器、构图原型 few-shot）在这个问题下还适用吗？** 如果适用，它们适用于哪些候选方案？
