# 画话本 / SayDraw — 前端设计规范
*视觉方向：暖纸草稿本（米色纸纹 + 黑墨线 + 一点暖橙）*
*配合《SayDraw design brief》一起用。目标设备：平板横屏优先，手机竖屏次之。*

---

## 0. 一句话基调
像一对父子窝在沙发上，在一本旧速写本里一起涂故事。**私密、手作、温暖、克制。** 墨线是主角，界面是配角。

---

## 1. 配色 token（可直接当 CSS variables）

```
--paper:        #f1ebdb;   /* 主背景，米色纸 */
--paper-card:   #fdfaf0;   /* 画板卡片，比背景亮一点 */
--ink:          #211e18;   /* 主墨色，偏暖的近黑，不要纯黑 */
--ink-soft:     rgba(33,30,24,.55);  /* 次要文字 */
--accent:       #d9622b;   /* 暖橙，仅用于点缀：胶纸/激活态/宝宝标记 */
--kid:          #d9622b;   /* 宝宝（暖） */
--dad:          #2b5d7e;   /* 爸爸（整套暖色里唯一的一抹冷，仅小面积点缀） */
```

铁律：**画面 SVG 里只有 `--ink` 一种颜色**，永不上色。暖橙越省着用越有效。

---

## 2. 字体

- **chrome / 标题 / 按钮**：`ZCOOL KuaiLe`（圆润、童趣、好认）。
- **旁白 / 故事文字**：`Ma Shan Zheng`（毛笔手写感，是"绘本的嗓音"，字号要大、行距要松）。
- **次要小字 / 占位符**：`Noto Serif SC` 或 `LXGW WenKai`（可读、安静）。
- 避免 Inter / Arial 这类没性格的字体。若出现英文，配 `Caveat` 之类手写拉丁体，保持手作感。

---

## 3. 纸纹与质感（这是"暖纸"成立的关键，别省）

- **纸纹**：低透明度的颗粒噪声叠在背景上（SVG `feTurbulence` 或一张噪声贴图，opacity 5%~8%），给"真纸"触感。
- **速写网格**：极淡的点阵（`radial-gradient` 点，间距 ~22px，opacity ~6%），像速写本的隐约方格。
- **深度**：所有"卡片"（画板、输入栏）用**偏硬的错位投影**（如 `6px 7px 0 rgba(33,30,24,.85)`），营造"贴在桌上的照片/便签"感，不要柔和的弥散阴影。
- **手绘边框**：卡片边框略不规则（可用 SVG 描边或 border + 轻微 rotate），别用规整圆角矩形。
- **胶带**：画板顶部一小条半透明暖橙"胶纸"（轻微倾斜），速写本招牌细节。

---

## 4. 逐部件设计

### 画板（Hero，屏幕主角）
- 居中卡片，`--paper-card` 底，手绘边框 + 错位投影，顶部胶纸。
- SVG 满宽、自适应高度，设 `max-height` 保证输入栏永远在屏内。
- 线条**自己画出来**（见 §5）。
- 下方一行旁白：`Ma Shan Zheng`，前面一个小圆点用说话人的颜色。
- 右下角极小的页码"第 N / M 格"，`--ink-soft`。
- （可选音效）画的时候配铅笔沙沙声。

### 输入栏（制作态，主要给家长操作）
- 一个大号圆角"药丸"，错位投影，和画板呼应。
- **麦克风按钮放最显眼处（左侧、大圆）**——语音是主角。聆听时变暖橙并轻微脉动。
- 文字输入框次要（给家长补字用），占位符随说话人变："宝宝说……（或者点麦克风）" / "爸爸说……"。
- "画出来"主按钮：墨色填充药丸、纸色字，是唯一的实心强按钮。

### 说话人切换（爸爸说 / 宝宝说）
- 两个**超大**描边药丸，激活时填充各自颜色（宝宝暖橙、爸爸冷蓝）。
- 故意做大做圆，让 4 岁小孩也能戳中。

### "接下来呢？"引导
- 幽灵按钮 + 火花图标。
- 返回的提示显示成一张**暖橙虚线便签**，是给家长照着念给孩子听的。

### 胶片条（成长中的绘本）
- 横向滚动的小画框，像贴着的一排照片/电影胶格。
- 当前格**抬起 + 墨色边框 + 错位投影**，其余压暗缩小；编号用说话人颜色。
- 新一格从右侧滑入并自动滚到底。

### 播放态（观看模式，给孩子的魔法时刻）
- **所有控件退场**，背景换成更深的暖牛皮纸色 + 轻微暗角，让中间那张亮卡片"发光"。
- 单张画居中、放大，重放"自己画出来"动效，大字旁白在下。
- 自动逐格推进（~4 秒/格），只留极简的上一格/下一格/关闭。
- 配铅笔沙沙音效。这是整个产品的情绪顶点。

---

## 5. 动效（把劲都使在一处）

- **顶配时刻 = 线条自己画出来**：用 `stroke-dasharray/dashoffset`，对 SVG 内各元素**错峰**触发（element 1、2、3… 依次延迟 0.15s），看起来像一只手在一笔笔画。时长 1.2~1.8s。其余动效全部克制，给它让路。
- **新一格落桌**：卡片带轻微缩放 + 极小旋转入场，像照片被丢到桌上。
- **加载态 = 不要转圈圈**：画一段"铅笔沙沙试笔"的潦草动效（几根游走的线 / 抖动的涂鸦），让等待像"画板在想怎么画"。对孩子非常关键。
- **按钮按压**：轻微下移 + 投影收缩，"贴纸被按下"的手感。
- 缓动用有机的 ease-out / 轻微回弹，别用机械线性。

---

## 6. 微文案（"做工"分也看文案是否用心）

- 全程是"温暖的、面向孩子的家长口吻"。
- 报错**绝不吓人**："画板打了个小盹，再说一次试试 ✏️"。
- 加载："画板正在沙沙画……"。
- 引导按钮："接下来呢？"。
- 文字尽量少，能用图标 + 声音就别用字。

---

## 7. 儿童可操作 / 防呆要点

- 点击目标够大（建议最小边 ≥ 56px），没有小控件。
- **没有"错"这个概念**：怎么戳都有正反馈，不出现会吓到孩子的失败态。
- "重来"等破坏性操作要有温和确认。
- 墨对纸高对比，保证可读。
- 给不识字的孩子加**声音反馈**（出画声、提示朗读）。
- 布局遵循"同屏只有一个主角"：画板占主，输入栏钉底，胶片是中间一条窄带。

---

## 8. 两态切换（统领整个布局状态）

- **制作态**：输入栏 + 说话人 + 提示都在（其实是家长在操作）。
- **观看态（播放）**：控件全退，画与字满屏，背景压深。
- 这两态的来回，本身就是产品的呼吸节奏，设计上要让切换有仪式感。
