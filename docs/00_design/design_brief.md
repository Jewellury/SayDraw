## 2. 一句话产品定义

给"不知道怎么陪小小孩玩"的家长：**你说一句、孩子说一句，画板把口述的故事实时画成黑白线条分镜，连起来就是你们共创的动画绘本**；同时它是一扇看见孩子脑子里在想什么的窗。

---

## 3. 目标用户与核心价值

- **用户**：有学龄前小孩的家长（作者本人有 4 岁和 1 岁两个儿子）。
- **痛点**：孩子不爱我的爱好，我不爱他的绘本，亲子时间容易各玩各的、或塞屏幕。
- **价值 1（钩子）**：一个谁都不独占、必须一起创造的活动。
- **价值 2（灵魂）**：4 岁孩子只能用他见过 / 经历过的现实来编故事，所以这同时是了解他在想什么、幼儿园过得怎样、锻炼表达的窗口。这一条是它比"又一个 AI 画图玩具"高一档的地方。

---

## 4. 核心交互闭环（MVP 必须跑通的那条线）

1. 屏幕中央是"画板"，进来时已有一格开场（陨石砸小恐龙）。
2. 轮流：选"爸爸说 / 宝宝说"，用**语音**或打字输入一句。
3. 提交 → 后端调 LLM，返回两样东西：一句**衔接旁白** + 一幅**黑白线条 SVG**。
4. 这一格以"**线条一笔一笔自己画出来**"的动效出现，旁白显示在下方。
5. 所有格子排成胶片条，可点回看；"播放故事"把它们连成一段动画。
6. 孩子卡住 → "接下来呢"按钮，让模型用爸爸的口吻问一句来引导他往下说。

## 5. 已经定下来的产品/设计决策（请保留，别让技术方案推翻）

- **黑白线条手绘风**：不是没空做彩色，是刻意选的——像作家或孩子在草稿本上涂，门槛低、更"魔法"，也和下一条绑定。
- **画面用 SVG，不用位图**：因为核心动效是"线条自己画出来"（靠 `stroke-dasharray` 动画），位图做不到。**这是美学和技术绑死的决策点，是产品的记忆点，优先保住。**
- **语音优先**：4 岁孩子是"说"不是"打字"，语音是产品成立的前提，不是加分项。
- **实时生成**：每说一句即时出画，延迟体验非常关键（孩子等不了）。
- **AI 一次产出"衔接旁白 + 画面"**：保证故事连贯，而不是一堆孤立的配图。

## 6. 留给 Codex 一起拍板的关键技术决策点（重点）

我把约束和我的倾向列出来，最终方案你们定。

- **必须有后端/serverless 放 LLM key（最重要）**：现有原型那种"前端直接 fetch 调模型、且不带 key"的写法，只在 Claude 的 artifact 沙盒里成立。真实部署里**绝对不能把 API key 放前端**，需要一个轻后端代理（如 Next.js API route / Vercel Function / Cloudflare Worker）来转发请求、藏 key。
- **框架与托管**：我倾向 **Next.js + Vercel**——自带 serverless function 放 key、一键出公开网址、本身就是标准 GitHub repo，正好满足"公开部署 + Novus 连 GitHub"。但开放给 Codex 评估其它栈（Remix、SvelteKit、Vite + 独立后端等）。
- **画面生成方式**：模型直出 SVG（保住"自己画出来"动效）vs 调图像模型出 PNG（更稳但丢动效、且黑白线条风不一定可控）。我倾向 **SVG**，需要和 Codex 讨论：prompt 怎么约束、服务端怎么校验 SVG 合法性、超时/截断怎么兜底、坏图 fallback。
- **用哪个模型**：本地原型用的是 Claude；真实部署用 Anthropic 还是 OpenAI 由你们定（看成本、延迟、SVG 质量、以及 Codex 更熟哪个）。建议**一次调用同时返回旁白 + SVG（结构化 JSON）**，减少往返、降延迟。
- **延迟与等待感**：一句到出画尽量控制在几秒内；考虑 streaming，或用 loading 动效（比如先画一个"沙沙画"的占位）兜住等待。
- **语音输入**：MVP 用浏览器 **Web Speech API（zh-CN）**最快，但兼容性有限（iOS/Safari 较弱）。是否上更稳的语音转写服务，和 Codex 权衡时间成本。
- **数据持久化**：MVP 可纯前端内存、不落库（够交）。但"宝宝的小本子"（攒原话 + 月度回顾）需要存储 + 轻账号，是 2 周内的拉伸目标，不是必须项。请 Codex 帮判断要不要现在就把数据层接口预留好。
- **可被 Novus 插桩**：把关键交互（提交一句、播放、点提示、回看某格）做成结构清晰、命名有意义的事件/路由，方便 Novus 自动识别埋点。

## 7. 范围（2 周，先保能交）
